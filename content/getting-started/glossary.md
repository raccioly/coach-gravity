# 📖 Glossary — Every Term in Plain English

> Bookmark this page. Come back whenever you see a word you don't recognize.

---

| Term | Plain English |
|---|---|
| **API** | A way for two programs to talk to each other. Like a waiter taking your order to the kitchen — you don't go to the kitchen yourself, the waiter (API) handles the communication. |
| **Authentication (Auth)** | Verifying who someone is. Logging in = authentication. |
| **AWS** | Amazon Web Services — a massive collection of cloud services. Think of it as renting a computer on the internet to host your website. |
| **Backend** | The part of an app users don't see — data storage, business logic, security. The kitchen of a restaurant. |
| **Branch** | A separate copy of your project for trying things. Like editing a copy of a document. |
| **Browser** | Chrome, Firefox, Edge, Safari — the app you use to visit websites. |
| **Browser Console** | A hidden panel in your browser (open with `F12`) that shows errors and messages. Red text = something broke. Screenshot the errors and share with Antigravity to get instant fixes. |
| **Build** | Converting your source code into a version optimized for the internet. Like compiling a manuscript into a printed book. |
| **Chrome Extension** | A small add-on for Google Chrome that gives Antigravity the ability to see and interact with your browser. Required for `/stage` testing. |
| **CLI** | Command Line Interface — using text commands instead of clicking buttons. Same as "terminal." |
| **CMS** | Content Management System — lets non-technical people update website content (text, images) without touching code. |
| **Commit** | Saving a snapshot of your project with a note about what changed. Like pressing "Save" in a document. |
| **Component** | A reusable piece of a user interface — a button, a card, a navigation bar. Like LEGO blocks. |
| **Configuration (Config)** | Settings that control how software behaves, without changing the actual code. |
| **CORS** | A security rule that controls which websites can talk to your backend. |
| **CRM** | Customer Relationship Management — a system for tracking interactions with customers or contacts. |
| **CSS** | The language that controls how a webpage looks — colors, fonts, spacing, layout. |
| **Database** | Where data is stored permanently. Like a spreadsheet, but much faster and more organized. |
| **Dependency** | A piece of code your project relies on that someone else wrote. Like ingredients in a recipe. |
| **Deploy** | Putting your app on the internet so other people can access it. |
| **DynamoDB** | Amazon's database service — stores data in the cloud. |
| **Endpoint** | A specific URL your backend responds to. Like a phone extension at a company. |
| **Environment Variable** | A secret setting stored outside the code — like a password stored in a safe instead of written on a sticky note. |
| **Framework** | A pre-built structure for building apps. Like a house frame — you add walls, paint, and furniture. |
| **Frontend** | The part of an app users see and interact with — screens, buttons, forms. The dining room of a restaurant. |
| **Git** | A tool that tracks every change to your project. Like Google Docs version history, but for code. |
| **GitHub** | An online platform for storing Git repositories. Like Google Drive for code projects. |
| **HTML** | The language that defines the structure of a webpage — headings, paragraphs, images, links. |
| **HTTP/HTTPS** | The protocol for sending web pages. HTTPS is the secure version (the padlock icon). |
| **JavaScript** | The programming language that makes websites interactive. Antigravity writes this for you. |
| **JSON** | A format for storing and sending data. Looks like `{"name": "John", "age": 30}`. |
| **Localhost** | Your computer pretending to be a website. Only you can see it. Address: `http://localhost:3000`. |
| **Middleware** | Code that runs between a request and a response — like a security checkpoint at an airport. |
| **Next.js** | A popular framework for building web applications. Used by Netflix, TikTok, and many others. |
| **Node.js** | A tool that lets your computer run JavaScript. Needed for building web apps. |
| **npm** | Node Package Manager — a tool for downloading and managing dependencies. Like an app store for code building blocks. |
| **Open Source** | Software whose code is publicly available. Anyone can see how it works. Free to use. |
| **Package** | Same as a dependency — a reusable piece of code you can add to your project. |
| **Port** | An "apartment number" for your computer. When you see `localhost:3000`, the `:3000` is the port. |
| **Production** | The live, public version of your app that real users access. |
| **Push** | Uploading your code changes from your computer to GitHub. |
| **RBAC** | Role-Based Access Control — different users have different permissions (admin can delete, viewer can only read). |
| **Repository (Repo)** | Your project folder with version history tracked by Git. |
| **REST API** | A standard way to build backends that respond to web requests. |
| **Route** | A URL path in your app. `/about` is the "about" route, `/dashboard` is the dashboard. |
| **Schema** | A definition of what data looks like — what fields exist well you need and what type they are. |
| **Server** | A computer (or service) that hosts your app and serves it to visitors. |
| **SLA** | Service Level Agreement — a promise about response times or availability. |
| **SQL** | A language for talking to databases — asking questions, adding data, updating records. |
| **SSL** | Security technology that encrypts data between browsers and servers. The padlock icon in your browser. |
| **Terminal** | A text-based way to give commands to your computer. Like texting your computer. |
| **TypeScript** | JavaScript with added type safety — catches errors before they happen. |
| **UI/UX** | User Interface (what it looks like) / User Experience (how it feels to use). |
| **URL** | Web address — like `https://myapp.com/dashboard`. |
| **Version Control** | Tracking every change to your project so you can undo, review, and collaborate. Git is the tool. |
| **Webhook** | An automatic notification from one service to another when something happens. |
