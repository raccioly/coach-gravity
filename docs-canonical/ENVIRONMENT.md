## Prerequisites

| Requirement | Version | Purpose |
|------------|---------|----------|
| Antigravity AI Agent | Latest | Interactive coaching engine — reads curriculum and guides learners |
| Node.js | 18+ | Required by DocGuard CLI for documentation quality validation |
| npm / npx | 10+ | Package runner for DocGuard CLI commands |
| Python | 3.8+ | Required by Spec Kit CLI (`specify` command) |
| uv | 0.10+ | Python tool manager for installing Spec Kit CLI |
| Git | 2.30+ | Version control (used in Phase 7 of the curriculum) |

## Environment Variables

Coach Gravity operates entirely from local Markdown files and AI agent configuration. The toolkit reads all settings from its file structure.

| Variable | Required | Default | Description | Where to Get |
|----------|----------|---------|-------------|-------------|
| PATH | ✅ Yes | System default | Ensures `docguard`, `specify`, and `node` commands are accessible | System environment |

## Configuration Files

| File | Purpose | Template |
|------|---------|----------|
| `.docguard.json` | DocGuard validator configuration (profile, project type, enabled validators) | Generated by `docguard init` |
| `.specify/init-options.json` | Spec Kit initialization options (AI agent, script type) | Generated by `specify init` |
| `.specify/extension-catalogs.yml` | Spec Kit extension catalog configuration | Generated by `specify extension catalog add` |
| `.agent/workflows/start.md` | Primary Antigravity workflow — the entry point for learner onboarding | Included in toolkit |

## Setup Steps

1. Unzip `coach_gravity.zip` into your working directory
2. Open the directory in your AI coding agent (Antigravity)
3. Type `/start` in the agent chat to begin the guided onboarding
4. Follow the phases in `getting-started/` sequentially (Phase 0 → Phase 8)

### Developer Setup (For Toolkit Authors)

1. Clone the repository
2. Install DocGuard CLI: `npm i -g docguard-cli`
3. Install Spec Kit CLI: `uv tool install specify-cli --from git+https://github.com/github/spec-kit.git`
4. Run quality checks: `docguard guard`
5. Check documentation score: `docguard score`
